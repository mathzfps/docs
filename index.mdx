Bem-vindo à documentação da **Paysure API** — a forma mais simples de integrar PIX no seu produto.

## O que a Paysure faz

- **Recebe pagamentos PIX** (cash-in) via QR Code dinâmico gerado por API.
- **Envia pagamentos PIX** (cash-out) para chaves DICT (CPF, CNPJ, email, phone, EVP).
- **Distribui splits** automaticamente — divida cada transação entre múltiplos destinatários (% configurável).
- **Notifica via webhook** todo evento relevante (`paid`, `refunded`, etc) na URL que você fornece.

## Conceitos-chave

<CardGroup cols={2}>
  <Card title="Valores em centavos" icon="coin">
    Toda quantia é em **centavos** (`value_cents`). R$ 1,00 = 100. R$ 99,90 = 9990.
  </Card>
  <Card title="external_id é seu ID" icon="fingerprint">
    Sempre envie um `external_id` único por transação. Garante idempotência se a request for repetida.
  </Card>
  <Card title="Webhooks são a verdade" icon="bell">
    A resposta HTTP 201 só diz que a transação foi *aceita*. O status final (`paid`/`refunded`) chega via webhook.
  </Card>
  <Card title="Splits em %" icon="chart-pie">
    Splits são definidos em percentual do valor bruto (`{value: 5}` = 5%).
  </Card>
</CardGroup>
